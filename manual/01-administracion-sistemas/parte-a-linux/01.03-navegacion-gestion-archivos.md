# ¿Dónde estoy?

El primer comando que debe conocer cualquier administrador es:

```bash
pwd
```

Significa:

```text
Print Working Directory
```

Muestra el directorio actual.

Ejemplo.

```text
/home/ana
```

---

# ¿Qué hay aquí?

Para listar el contenido de un directorio utilizamos:

```bash
ls
```

Opciones muy habituales.

```bash
ls -l
```

Formato largo.

---

```bash
ls -a
```

Incluye archivos ocultos.

---

```bash
ls -lh
```

Tamaño legible.

---

```bash
ls -la
```

La combinación más utilizada.

---

# Cambiar de directorio

```bash
cd
```

Ejemplos.

Entrar en un directorio.

```bash
cd Documentos
```

---

Volver al directorio personal.

```bash
cd
```

o

```bash
cd ~
```

---

Subir un nivel.

```bash
cd ..
```

---

Ir al directorio raíz.

```bash
cd /
```

---

# Crear directorios

```bash
mkdir proyecto
```

Crear varios.

```bash
mkdir datos scripts resultados
```

Crear directorios anidados.

```bash
mkdir -p proyecto/datos/raw
```

---

# Crear archivos

```bash
touch informe.txt
```

También sirve para actualizar la fecha de modificación de un archivo existente.

---

# Copiar

```bash
cp origen destino
```

Ejemplo.

```bash
cp informe.txt copia.txt
```

Copiar directorios.

```bash
cp -r proyecto respaldo
```

---

# Mover o renombrar

```bash
mv informe.txt informes/
```

Renombrar.

```bash
mv informe.txt informe_final.txt
```

---

# Eliminar

Archivo.

```bash
rm informe.txt
```

Directorio vacío.

```bash
rmdir pruebas
```

Directorio con contenido.

```bash
rm -r proyecto
```

⚠️ Mucho cuidado.

Linux no envía los archivos a la papelera.

---

# Visualizar archivos

Mostrar todo.

```bash
cat informe.txt
```

---

Primeras líneas.

```bash
head informe.txt
```

---

Últimas líneas.

```bash
tail informe.txt
```

Muy útil para leer logs.

---

Visualización página a página.

```bash
less informe.txt
```

Pulsa **q** para salir.

---

# Buscar archivos

Buscar por nombre.

```bash
find /home -name "*.txt"
```

Buscar una carpeta.

```bash
find . -type d
```

---

# Linux en acción

El administrador te pide preparar esta estructura.

```text
proyecto

├── datos

├── scripts

└── resultados
```

Podrías hacerlo con:

```bash
mkdir -p proyecto/{datos,scripts,resultados}
```

Ahora crea un archivo.

```bash
touch proyecto/scripts/proceso.sh
```

Haz una copia.

```bash
cp proyecto/scripts/proceso.sh proyecto/scripts/proceso.bak
```

Renómbralo.

```bash
mv proyecto/scripts/proceso.bak proyecto/scripts/proceso_v2.sh
```

# ⚠️ Errores frecuentes

❌ Confundir `cp` con `mv`.

---

❌ Utilizar `rm -r` sin comprobar el directorio.

---

❌ Olvidar `-p` al crear directorios anidados.

---

❌ Pensar que Linux tiene papelera.

**R:** `ls -a`

---

**P:** ¿Qué hace `cd ..`?

**R:** Sube un nivel en la jerarquía de directorios.
