# ¿Qué es GNU/Linux?

Existe una confusión muy habitual.

❌ **Incorrecto**

> Linux es el sistema operativo.

✅ **Correcto**

- Linux = Kernel (núcleo).
- GNU = herramientas del sistema.
- GNU/Linux = sistema operativo completo.

# ¿Qué es el Kernel?

El Kernel es el componente más importante del sistema operativo.

Se encarga de comunicar el software con el hardware y de gestionar los recursos del sistema.

Entre sus funciones destacan:

| Función | Ejemplo |
|---------|---------|
| Gestión de procesos | Crear y finalizar procesos |
| Gestión de memoria | Administración de la RAM |
| Sistema de archivos | ext4, XFS... |
| Dispositivos | USB, discos, impresoras |
| Redes | TCP/IP |

💡 Sin Kernel ningún programa podría ejecutarse.

---

# ¿Qué aporta GNU?

GNU proporciona la mayoría de utilidades que utilizamos diariamente.

Por ejemplo:

- ls
- cp
- mv
- rm
- cat
- chmod
- chown

Curiosamente, muchos de los llamados "comandos Linux" pertenecen realmente al proyecto GNU.

---

# ¿Qué es la Shell?

La Shell es el programa que interpreta los comandos escritos por el usuario.

Ejemplo:

```bash
ls
```

Cuando ejecutamos ese comando ocurre lo siguiente:

1. La Shell lee el comando.
2. Busca el programa correspondiente.
3. Solicita al Kernel su ejecución.
4. El Kernel realiza la operación.
5. La Shell muestra el resultado.

La Shell más utilizada en GNU/Linux es **Bash (Bourne Again SHell)**.

También existen otras alternativas:

- sh
- zsh
- fish

Para esta oposición, Bash será la más importante.

---

# Linux en acción

Imagina que escribes:

```bash
ls
```

El recorrido sería:

```text
Usuario

↓

Bash interpreta el comando

↓

GNU ejecuta la utilidad "ls"

↓

El Kernel solicita acceso al sistema de archivos

↓

El hardware responde

↓

Bash muestra el resultado
```

Aunque este proceso sucede en milésimas de segundo, comprenderlo ayuda a entender cómo funciona internamente un sistema Linux.

# ¿Qué es una distribución?

Una distribución reúne todos los componentes necesarios para formar un sistema operativo completo.

Normalmente incluye:

- Kernel Linux
- Herramientas GNU
- Gestor de paquetes
- Instalador
- Aplicaciones
- Configuración del sistema

Ejemplos:

- Debian
- Ubuntu
- Fedora
- Red Hat Enterprise Linux (RHEL)
- Rocky Linux
- AlmaLinux

---

# Familias de distribuciones

No es necesario memorizar decenas de distribuciones.

Basta con conocer las dos familias más importantes.

| Familia | Gestor de paquetes |
|----------|-------------------|
| Debian / Ubuntu | apt |
| Red Hat / Rocky / AlmaLinux | dnf (antes yum) |

Estas familias aparecerán repetidamente durante el resto del temario.

# ¿Por qué el IMIDA utiliza Linux?

Linux está ampliamente implantado en organismos públicos y centros de investigación debido a:

- estabilidad;
- seguridad;
- flexibilidad;
- alto rendimiento;
- ausencia de costes de licencia;
- enorme comunidad de desarrollo.

Por ello constituye la plataforma habitual para servidores, bases de datos y servicios científicos.

# ⚠️ Errores frecuentes

❌ Ubuntu = Linux.

✔ Ubuntu es una distribución GNU/Linux.

---

❌ Bash = Linux.

✔ Bash es una Shell.

---

❌ Linux = sistema operativo completo.

✔ Linux es el Kernel.

# 📝 Resumen

GNU/Linux constituye la base de la mayoría de servidores modernos.

En este capítulo has aprendido que:

- Linux es el Kernel.
- GNU proporciona las herramientas del sistema.
- Bash es la Shell más utilizada.
- Una distribución combina Kernel, GNU y aplicaciones.
- Debian y Red Hat representan las dos grandes familias de distribuciones.

Este conocimiento servirá como base para comprender el resto del bloque de Administración de Sistemas.
