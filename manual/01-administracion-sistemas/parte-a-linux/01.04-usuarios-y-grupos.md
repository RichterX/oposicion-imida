# ¿Por qué existen los usuarios?

En un servidor pueden trabajar decenas o incluso cientos de personas.

Linux necesita:

- identificar quién realiza cada acción;
- controlar permisos;
- proteger la información;
- registrar cambios;
- impedir que un usuario interfiera con otro.

🎯 **Idea clave**

Un usuario representa una identidad dentro del sistema operativo.

---

# 🧠 ¿Por qué existe esto?

Imagina que dos investigadores modifican el mismo archivo.

¿Cómo sabe Linux quién realizó el cambio?

No utiliza únicamente el nombre del usuario.

Internamente utiliza un identificador numérico llamado **UID**.

Ese identificador nunca cambia, aunque el nombre del usuario pueda modificarse.

Por eso Linux trabaja realmente con números y no con nombres.

---

# Usuario vs UID

Cada usuario tiene asociado un identificador único denominado:

```text
UID
```

Ejemplo:

| Usuario | UID |
|---------|----:|
| root | 0 |
| ana | 1001 |
| carlos | 1002 |

🎯 **Importante**

Linux identifica realmente a los usuarios mediante el **UID**, no mediante su nombre.

---

# ¿Qué es un grupo?

Un grupo permite reunir varios usuarios para compartir permisos.

Ejemplo.

Grupo:

```text
investigacion
```

Usuarios:

- ana
- carlos
- maria

Gracias a los grupos no es necesario asignar permisos individualmente a cada usuario.

---

# GID (Group Identifier)

Al igual que los usuarios poseen un UID, los grupos disponen de un identificador numérico denominado:

```text
GID
```

Ejemplo.

| Grupo | GID |
|--------|----:|
| root | 0 |
| investigacion | 1002 |
| postgres | 26 |

Linux también identifica internamente los grupos mediante su GID.

---

# Grupo principal y grupos secundarios

Cada usuario posee:

- un **grupo principal**;
- cero o más **grupos secundarios**.

Ejemplo.

Usuario:

```text
ana
```

Grupo principal:

```text
ana
```

Grupos secundarios:

```text
investigacion

docker

postgres
```

Esto permite que un mismo usuario pueda acceder a distintos recursos sin duplicar permisos.

---

# El usuario root

El usuario más importante del sistema es:

```text
root
```

Características:

- UID = 0.
- Control prácticamente total del sistema.
- Puede modificar cualquier archivo.

⚠️ No debe confundirse:

```text
/
```

Directorio raíz del sistema.

Con:

```text
/root
```

Directorio personal del usuario `root`.

---

# 💻 Linux en acción

Consultar el usuario actual.

```bash
whoami
```

Ejemplo.

```text
ana
```

---

Consultar información completa del usuario.

```bash
id
```

Salida.

```text
uid=1001(ana)

gid=1001(ana)

groups=1001(ana),1002(investigacion)
```

Esta información indica:

- UID.
- Grupo principal.
- Grupos secundarios.

---

Consultar otro usuario.

```bash
id postgres
```

# 📝 Resumen

Linux necesita identificar de forma inequívoca quién realiza cada acción.

Para ello utiliza:

- **UID**, que identifica a los usuarios.
- **GID**, que identifica a los grupos.

Los grupos permiten compartir permisos entre varios usuarios y simplifican enormemente la administración del sistema.
