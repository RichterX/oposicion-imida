# ¿Cómo organiza Linux sus archivos?

A diferencia de Windows, Linux organiza toda la información en **un único árbol de directorios**.

Todo comienza en un único punto:

```text
/
```

Este directorio recibe el nombre de **directorio raíz (root)**.

Todos los demás directorios cuelgan de él.

```text
/
├── bin
├── boot
├── dev
├── etc
├── home
├── media
├── mnt
├── opt
├── proc
├── root
├── run
├── srv
├── tmp
├── usr
└── var
```

🎯 **Idea clave**

En Linux todo forma parte del mismo árbol.

No existen unidades independientes como `C:` o `D:`.

---

# Linux vs Windows

| Windows | Linux |
|----------|--------|
| C:\ | / |
| D:\ | Todo pertenece al mismo árbol |
| Unidades independientes | Sistema jerárquico único |

Esta diferencia es importante porque muchos usuarios procedentes de Windows esperan encontrar varias unidades de disco, algo que normalmente no ocurre en Linux.

# Directorios imprescindibles

## /etc ⭐⭐⭐⭐⭐

Contiene la configuración del sistema y de la mayoría de servicios.

Ejemplos:

- PostgreSQL
- Apache
- SSH
- Cron

🎯 Si buscas un fichero de configuración, piensa primero en **/etc**.

---

## /home ⭐⭐⭐⭐⭐

Contiene los directorios personales de los usuarios.

Ejemplo:

```text
/home/ana
```

Aquí suelen almacenarse:

- documentos;
- descargas;
- scripts;
- configuración personal.

---

## /var ⭐⭐⭐⭐⭐

Contiene información que cambia constantemente.

Ejemplos:

- logs (`/var/log`);
- colas de impresión;
- cachés;
- correo;
- bases de datos de algunos servicios.

🎯 Cuando un servicio falla, una de las primeras comprobaciones suele hacerse en **/var/log**.

---

## /usr ⭐⭐⭐⭐☆

Contiene la mayor parte de programas, bibliotecas y utilidades del sistema.

No significa "usuario".

Históricamente significa:

> **Unix System Resources**

---

## /tmp ⭐⭐⭐⭐☆

Archivos temporales.

Su contenido puede eliminarse automáticamente al reiniciar el sistema o periódicamente.

Nunca almacenes aquí información importante.

---

## /dev ⭐⭐⭐☆☆

Representa los dispositivos del sistema como si fueran archivos.

Ejemplos:

- discos;
- terminales;
- USB;
- particiones.

---

## /proc ⭐⭐⭐☆☆

Sistema de archivos virtual.

No almacena datos reales en disco.

Permite consultar información sobre:

- procesos;
- memoria;
- CPU;
- Kernel.

---

## /root ⭐⭐⭐☆☆

Directorio personal del administrador (**root**).

⚠️ No debe confundirse con:

```text
/
```

que representa el directorio raíz del sistema.

---

# Linux en acción

Imagina que PostgreSQL deja de funcionar.

¿Qué directorios revisarías?

1. Configuración:

```text
/etc/postgresql
```

2. Registros:

```text
/var/log
```

3. Información del sistema:

```text
/proc
```

Conocer esta estructura permite localizar problemas mucho más rápidamente.

# Resumen visual

| Directorio | Contenido principal |
|------------|---------------------|
| / | Raíz del sistema |
| /etc | Configuración |
| /home | Usuarios |
| /var | Logs y datos variables |
| /usr | Programas |
| /tmp | Archivos temporales |
| /dev | Dispositivos |
| /proc | Información del sistema |
| /root | Directorio del administrador |

---

# 🧠 Mapa mental

```text
/

├── etc → Configuración

├── home → Usuarios

├── var → Logs

├── usr → Programas

├── tmp → Temporales

├── proc → Sistema

├── dev → Dispositivos

└── root → Administrador
```

---

# ⚠️ Errores frecuentes

❌ `/root` es el directorio raíz.

✔ `/` es el directorio raíz.

`/root` es únicamente el directorio personal del administrador.

---

❌ `/usr` contiene los usuarios.

✔ Contiene programas y recursos del sistema.

---

❌ `/proc` almacena archivos normales.

✔ Es un sistema de archivos virtual generado por el Kernel.

**P:** ¿Cuál es el directorio raíz del sistema?

# 📝 Resumen

Linux organiza toda la información en un único árbol cuyo origen es el directorio raíz (`/`).

En este capítulo has aprendido a identificar los directorios más importantes y su función:

- `/etc` almacena la configuración del sistema.
- `/home` contiene los directorios personales de los usuarios.
- `/var` almacena datos variables y registros.
- `/usr` contiene programas y bibliotecas.
- `/proc` ofrece información virtual sobre el sistema.
- `/dev` representa los dispositivos del equipo.

En el siguiente capítulo aprenderás a desplazarte por este árbol y a trabajar con archivos y directorios mediante la línea de comandos.
