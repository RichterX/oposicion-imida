# ¿Qué es un servicio?

Un servicio es un programa que se ejecuta en segundo plano y proporciona una función al sistema.

Ejemplos.

- Servidor web Apache.
- Base de datos PostgreSQL.
- Servidor SSH.
- Servicio de impresión.

La mayoría de ellos comienzan automáticamente al arrancar el sistema.

---

# ¿Qué es systemd?

En la mayoría de distribuciones Linux actuales, el encargado de gestionar los servicios es:

```text
systemd
```

Es el sistema de inicio (*init system*) más utilizado.

Entre sus funciones se encuentran:

- Iniciar servicios.
- Detener servicios.
- Reiniciarlos.
- Supervisarlos.
- Gestionar el arranque del sistema.

🎯 **Idea clave**

> `systemd` administra los servicios del sistema.

---

# Linux en acción

Consultar el estado de PostgreSQL.

```bash
systemctl status postgresql
```

Ejemplo de salida:

```text
● postgresql.service

Active: active (running)

Loaded: loaded
```

Si aparece:

```text
active (running)
```

El servicio está funcionando correctamente.

---

# Iniciar un servicio

```bash
sudo systemctl start postgresql
```

---

# Detener un servicio

```bash
sudo systemctl stop postgresql
```

---

# Reiniciar un servicio

```bash
sudo systemctl restart postgresql
```

Muy utilizado después de modificar un fichero de configuración.

---

# Recargar la configuración

En algunos servicios no es necesario reiniciar completamente.

```bash
sudo systemctl reload apache2
```

Esto vuelve a cargar la configuración sin detener el servicio.

---

# Consultar si un servicio está activo

```bash
systemctl is-active postgresql
```

Salida posible.

```text
active
```

---

# Arranque automático

Para activar un servicio al iniciar el sistema.

```bash
sudo systemctl enable postgresql
```

Para desactivarlo.

```bash
sudo systemctl disable postgresql
```

---

# Consultar todos los servicios

```bash
systemctl list-units --type=service
```

Obtendremos una lista con todos los servicios activos.

---

# journalctl

Los servicios generan registros.

Podemos consultarlos mediante:

```bash
journalctl
```

Ver solo PostgreSQL.

```bash
journalctl -u postgresql
```

Ver las últimas líneas.

```bash
journalctl -u postgresql -n 20
```

Seguir los registros en tiempo real.

```bash
journalctl -u postgresql -f
```

Muy útil cuando estamos buscando errores.

# Linux en acción

La aplicación web deja de funcionar.

Primer paso.

```bash
systemctl status apache2
```

Si aparece:

```text
inactive
```

Lo iniciamos.

```bash
sudo systemctl start apache2
```

Comprobamos de nuevo.

```bash
systemctl status apache2
```

Ahora aparece.

```text
active (running)
```

Problema resuelto.

# Errores frecuentes

❌ Reiniciar un servicio sin comprobar antes su estado.

---

❌ Confundir `restart` con `reload`.

---

❌ Olvidar habilitar un servicio con `enable`.

Tras reiniciar el servidor dejará de iniciarse automáticamente.
