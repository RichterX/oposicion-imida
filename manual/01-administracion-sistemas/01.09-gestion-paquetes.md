# ¿Qué es un paquete?

Un paquete es un archivo que contiene todo lo necesario para instalar un programa.

Normalmente incluye:

- Ejecutables.
- Bibliotecas.
- Archivos de configuración.
- Documentación.
- Información sobre dependencias.

Ejemplos de paquetes:

- postgresql
- apache2
- openssh-server
- git
- htop

🎯 **Idea clave**

> Un paquete es la unidad de instalación de software en Linux.

---

# ¿Qué es un gestor de paquetes?

Un gestor de paquetes es la herramienta encargada de:

- instalar programas;
- actualizarlos;
- eliminarlos;
- resolver dependencias automáticamente.

En Debian y Ubuntu el gestor más habitual es:

```text
APT
```

---

# Repositorios

Los paquetes no suelen descargarse desde páginas web.

Se obtienen desde servidores oficiales llamados:

```text
Repositorios
```

El gestor de paquetes consulta esos repositorios para saber:

- qué programas existen;
- qué versiones hay disponibles;
- qué actualizaciones existen.

---

# Linux en acción

Actualizar la lista de paquetes disponibles.

```bash
sudo apt update
```

⚠️ Importante.

Este comando **NO actualiza el sistema**.

Solo descarga la información más reciente de los repositorios.

---

# Actualizar los programas

Una vez actualizada la información...

Podemos instalar las nuevas versiones.

```bash
sudo apt upgrade
```

🎯 **Muy preguntable**

| Comando | Función |
|----------|----------|
| `apt update` | Actualiza la lista de paquetes |
| `apt upgrade` | Actualiza los paquetes instalados |

---

# Instalar un programa

Ejemplo.

```bash
sudo apt install htop
```

APT:

- descarga el paquete;
- instala dependencias;
- configura el programa.

Todo automáticamente.

---

Otro ejemplo.

```bash
sudo apt install postgresql
```

---

# Eliminar un programa

```bash
sudo apt remove htop
```

El programa desaparece.

Pero pueden permanecer algunos archivos de configuración.

---

# Eliminar completamente

```bash
sudo apt purge htop
```

Además del programa elimina también sus archivos de configuración.

---

# Eliminar dependencias innecesarias

Después de desinstalar programas...

Pueden quedar paquetes que ya no utiliza nadie.

Para eliminarlos.

```bash
sudo apt autoremove
```

Muy recomendable de vez en cuando.

---

# Buscar paquetes

Antes de instalar algo...

Podemos buscarlo.

```bash
apt search postgresql
```

---

# Mostrar información de un paquete

```bash
apt show postgresql
```

Obtendremos:

- versión;
- descripción;
- dependencias;
- tamaño.

# Linux en acción

Necesitamos instalar Git.

Primero actualizamos la información.

```bash
sudo apt update
```

Después instalamos.

```bash
sudo apt install git
```

Comprobamos la versión.

```bash
git --version
```

Resultado.

Programa listo para utilizar.

---

# Errores frecuentes

❌ Pensar que `apt update` actualiza el sistema.

No.

Solo actualiza la información de los repositorios.

---

❌ Olvidar ejecutar `apt update` antes de instalar software.

---

❌ Confundir `remove` con `purge`.

---

❌ Descargar programas desde Internet cuando existen en los repositorios oficiales.
