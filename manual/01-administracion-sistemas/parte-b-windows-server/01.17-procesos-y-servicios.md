# 🧠 ¿Por qué existen los procesos y los servicios?

Un sistema operativo debe ejecutar simultáneamente:

- aplicaciones de usuario;
- servicios de red;
- procesos internos;
- tareas programadas.

Para organizar todo este trabajo Windows distingue entre **procesos** y **servicios**.

---

# ¿Qué es un proceso?

Un proceso es un programa que se está ejecutando.

Cada proceso dispone de:

- identificador (PID);
- memoria propia;
- prioridad;
- recursos asignados.

Ejemplos:

- explorer.exe
- notepad.exe
- powershell.exe
- sqlservr.exe

---

# ¿Qué es un servicio?

Un servicio es un proceso diseñado para ejecutarse en segundo plano.

Normalmente:

- arranca automáticamente;
- no necesita un usuario conectado;
- proporciona funciones al sistema o a otros equipos.

Ejemplos:

- Windows Update
- DNS Server
- DHCP Server
- Print Spooler

---

# Administrador de tareas

La herramienta más conocida es:

```text
Administrador de tareas
```

Permite:

- visualizar procesos;
- comprobar uso de CPU;
- memoria;
- disco;
- red;
- finalizar procesos.

Es el equivalente más cercano a `top` o `htop` en GNU/Linux.

---

# Servicios

La consola:

```text
services.msc
```

permite:

- iniciar servicios;
- detener servicios;
- reiniciarlos;
- configurar el tipo de inicio;
- consultar su estado.

---

# Service Control Manager (SCM)

Windows administra todos los servicios mediante el:

```text
Service Control Manager
```

(SCM)

Este componente controla:

- inicio;
- parada;
- reinicio;
- dependencias;
- estado.

Es el equivalente conceptual a **systemd** en GNU/Linux.

---

# PowerShell

Consultar procesos:

```powershell
Get-Process
```

Consultar un proceso concreto:

```powershell
Get-Process explorer
```

Finalizar un proceso:

```powershell
Stop-Process
```

Consultar servicios:

```powershell
Get-Service
```

Iniciar un servicio:

```powershell
Start-Service
```

Detener un servicio:

```powershell
Stop-Service
```

Reiniciar un servicio:

```powershell
Restart-Service
```

---

# 🐧 Linux vs 🪟 Windows

| GNU/Linux | Windows |
|------------|----------|
| ps | Get-Process |
| top / htop | Administrador de tareas |
| kill | Stop-Process |
| systemctl | Services.msc / SCM |
| systemctl restart | Restart-Service |
