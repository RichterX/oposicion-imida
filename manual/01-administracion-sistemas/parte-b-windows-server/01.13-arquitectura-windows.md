# 🧠 ¿Por qué existe esta arquitectura?

Windows debe ser capaz de:

- ejecutar aplicaciones;
- administrar miles de procesos;
- controlar el hardware;
- gestionar memoria;
- proporcionar seguridad;
- ofrecer estabilidad incluso cuando una aplicación falla.

Para conseguirlo divide el sistema operativo en distintas capas con responsabilidades muy definidas.

Esta separación mejora:

- la estabilidad;
- la seguridad;
- el rendimiento;
- la facilidad de mantenimiento.

---

# Arquitectura general de Windows

De forma simplificada, Windows puede representarse mediante el siguiente esquema:

```text
Aplicaciones

↓

Servicios

↓

Windows Executive

↓

Kernel

↓

HAL (Hardware Abstraction Layer)

↓

Hardware
```

Cada capa se comunica únicamente con la inmediatamente inferior.

Este diseño evita que las aplicaciones accedan directamente al hardware.

---

# Componentes principales

## Aplicaciones

Son los programas que utilizan los usuarios.

Ejemplos:

- Microsoft Word
- Google Chrome
- Microsoft Edge
- SQL Server Management Studio

Las aplicaciones funcionan normalmente en **User Mode**.

---

## Servicios

Los servicios son programas especiales que trabajan en segundo plano.

Ejemplos:

- Windows Update
- Servicio DNS
- DHCP
- Print Spooler

Aunque no exista ningún usuario conectado, continúan ejecutándose.

Más adelante dedicaremos un capítulo completo a ellos.

---

## Windows Executive

Es el conjunto de componentes responsables de administrar:

- memoria;
- procesos;
- seguridad;
- entrada/salida;
- objetos del sistema.

Puede considerarse el "centro de operaciones" del sistema operativo.

---

## Kernel

El Kernel es el núcleo del sistema operativo.

Entre sus funciones destacan:

- planificación de procesos;
- gestión de memoria;
- sincronización;
- comunicación con los controladores;
- seguridad del sistema.

Si el Kernel deja de funcionar correctamente, Windows puede detenerse por completo.

---

## HAL (Hardware Abstraction Layer)

La HAL actúa como intermediaria entre Windows y el hardware.

Gracias a ella:

- Windows no necesita conocer cada dispositivo físico en detalle;
- los fabricantes desarrollan controladores compatibles;
- el sistema puede ejecutarse sobre hardware muy diverso.

---

# User Mode y Kernel Mode

Windows divide la ejecución en dos modos.

## User Mode

Aquí se ejecutan:

- aplicaciones;
- utilidades;
- procesos de usuario.

Características:

- acceso limitado;
- no puede acceder directamente al hardware;
- un fallo normalmente solo afecta a esa aplicación.

---

## Kernel Mode

Aquí trabajan:

- Kernel;
- controladores;
- componentes críticos.

Características:

- acceso completo al hardware;
- máximo nivel de privilegios;
- un error puede provocar un fallo completo del sistema (BSOD).

---

# Comparativa

| User Mode | Kernel Mode |
|------------|-------------|
| Aplicaciones | Núcleo del sistema |
| Acceso limitado | Acceso total |
| Más seguro | Máximo privilegio |
| Fallo aislado | Puede afectar a todo el sistema |

# El Registro de Windows

Una de las mayores diferencias respecto a Linux es la forma de almacenar la configuración.

En Linux:

```text
/etc
```

En Windows:

```text
Registry (Registro)
```

El Registro es una base de datos jerárquica que almacena:

- configuración del sistema;
- usuarios;
- aplicaciones;
- hardware;
- servicios.

Las principales ramas son:

- HKEY_LOCAL_MACHINE
- HKEY_CURRENT_USER
- HKEY_CLASSES_ROOT
- HKEY_USERS
- HKEY_CURRENT_CONFIG

No es necesario memorizar todas en profundidad para esta oposición, pero conviene conocerlas.

Las más utilizadas son:

- Server Manager

# 🐧 Linux vs 🪟 Windows

| GNU/Linux | Windows Server |
|------------|----------------|
| Kernel Linux | Windows NT Kernel |
| /etc | Registro |
| systemd | Service Control Manager |
| journalctl | Event Viewer |
| ps | Task Manager |
| Bash | PowerShell |
| ext4 | NTFS |

🎯 Ambos sistemas operativos persiguen el mismo objetivo: administrar recursos y ofrecer servicios de forma segura.

# ⚠️ Errores frecuentes

❌ Pensar que el Registro son archivos de texto.

✔ Es una base de datos jerárquica.

---

❌ Pensar que todos los procesos son servicios.

✔ Un servicio es un proceso diseñado para ejecutarse en segundo plano.

---

❌ Creer que User Mode y Kernel Mode tienen los mismos privilegios.

✔ Kernel Mode dispone de acceso completo al sistema.
