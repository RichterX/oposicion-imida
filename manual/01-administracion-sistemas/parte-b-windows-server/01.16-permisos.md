# 🧠 ¿Por qué existen las ACL?

En un servidor trabajan numerosos usuarios.

No todos deben poder:

- leer;
- modificar;
- eliminar;
- ejecutar;
- cambiar permisos.

Las ACL permiten controlar con precisión qué puede hacer cada usuario sobre cada recurso.

---

# ¿Qué es una ACL?

ACL significa:

```text
Access Control List
```

(Lista de Control de Acceso)

Cada archivo y carpeta posee una ACL.

En ella aparecen:

- usuarios;
- grupos;
- permisos asociados.

Cuando un usuario intenta acceder a un recurso, Windows consulta su ACL.

---

# Permisos habituales

Los permisos básicos más utilizados son:

| Permiso | Función |
|----------|----------|
| Control total | Acceso completo |
| Modificar | Leer, escribir y eliminar |
| Leer y ejecutar | Lectura y ejecución |
| Mostrar contenido | Ver carpetas |
| Lectura | Solo lectura |
| Escritura | Crear y modificar archivos |

---

# Permisos explícitos y heredados

Un permiso puede ser:

## Explícito

Asignado directamente al recurso.

---

## Heredado

Procedente de la carpeta superior.

Ejemplo.

```text
D:\Investigacion

↓

ProyectoA

↓

datos.xlsx
```

Si ProyectoA hereda permisos, los archivos también los heredarán.

La herencia simplifica enormemente la administración.

---

# Permisos efectivos

Un usuario puede pertenecer a varios grupos.

Ejemplo.

```text
Ana

↓

Investigacion

↓

Administrativos
```

Windows combina todos los permisos y obtiene el resultado final.

A este resultado se le denomina:

> **Permisos efectivos**

---

# Permitir y Denegar

Cada permiso puede aparecer como:

✅ Permitir

o

❌ Denegar

---

⚠️ Regla importante

Cuando existe un permiso **Denegar** explícito, normalmente prevalece sobre los permisos **Permitir**.

Por ello debe utilizarse únicamente cuando sea realmente necesario.

---

# El propietario

Todo archivo posee un propietario.

El propietario puede:

- modificar permisos;
- cambiar el propietario;
- administrar el recurso.

Aunque otro usuario tenga permisos elevados, el propietario conserva un papel especial sobre el objeto.

---

# Buenas prácticas

✔ Asignar permisos a grupos.

✔ Evitar asignarlos directamente a usuarios.

✔ Utilizar la herencia siempre que sea posible.

✔ Utilizar "Denegar" solo cuando sea imprescindible.

✔ Aplicar el principio del mínimo privilegio.

---

# 🐧 Linux vs 🪟 Windows

| GNU/Linux | Windows Server |
|------------|----------------|
| chmod | ACL |
| chown | Propietario |
| Grupos | Grupos |
| Herencia limitada | Herencia automática |
| rwx | Permisos detallados |

🎯 Windows ofrece un modelo de permisos más granular, mientras que Linux utiliza un sistema más sencillo basado en propietario, grupo y otros.
