# 🧠 ¿Por qué es necesario el mantenimiento?

Un servidor permanece funcionando durante meses o incluso años.

Durante ese tiempo:

- aparecen nuevas vulnerabilidades;
- se instalan aplicaciones;
- crecen los registros;
- aumenta el uso del disco;
- cambian los usuarios;
- evolucionan los servicios.

Sin mantenimiento, incluso un servidor correctamente configurado terminará presentando problemas.

---

# Tareas habituales de administración

Las tareas más frecuentes son:

- comprobar el estado general del servidor;
- revisar el Visor de eventos;
- instalar actualizaciones;
- verificar copias de seguridad;
- comprobar espacio en disco;
- revisar servicios;
- controlar el rendimiento;
- administrar usuarios y permisos.

Estas tareas forman parte del mantenimiento preventivo.

---

# Administración remota

En muchas ocasiones el administrador no trabaja físicamente delante del servidor.

Windows permite administrarlo de forma remota mediante distintas herramientas.

## Escritorio remoto (RDP)

Permite acceder gráficamente al servidor desde otro equipo.

Es una de las herramientas más utilizadas en entornos Windows.

Ventajas:

- administración completa;
- interfaz gráfica;
- acceso remoto seguro (si está correctamente configurado).

---

## PowerShell Remoting

PowerShell también permite administrar servidores remotamente mediante comandos.

Ejemplo:

```powershell
Enter-PSSession NombreServidor
```

o

```powershell
Invoke-Command
```

Resulta especialmente útil para automatizar tareas en varios servidores.

---

# Visor de eventos

Una de las herramientas más importantes es:

```text
Event Viewer
```

Permite consultar:

- errores;
- advertencias;
- información del sistema;
- eventos de seguridad;
- registros de aplicaciones.

Cuando aparece un problema, suele ser uno de los primeros lugares que debe revisarse.

---

# Windows Update

Mantener el sistema actualizado es una tarea fundamental.

Las actualizaciones permiten:

- corregir errores;
- solucionar vulnerabilidades;
- mejorar la estabilidad;
- incorporar nuevas funcionalidades.

Como norma general:

✔ instalar únicamente actualizaciones verificadas.

✔ planificar reinicios fuera del horario de producción.

---

# Copias de seguridad

Ningún servidor está completamente protegido frente a:

- fallos de hardware;
- errores humanos;
- ransomware;
- corrupción de datos.

Por ello deben realizarse copias de seguridad periódicas.

Windows Server incorpora la característica:

```text
Windows Server Backup
```

Las copias deben verificarse periódicamente.

Una copia no comprobada no puede considerarse una copia válida.

---

# Monitorización

Administrar un servidor también implica observar su comportamiento.

Conviene revisar periódicamente:

- CPU;
- memoria;
- almacenamiento;
- utilización de red;
- servicios críticos.

Una degradación progresiva suele detectarse antes mediante monitorización que esperando a que aparezca un fallo.

---

# Programador de tareas

Muchas tareas administrativas pueden automatizarse.

Windows incorpora:

```text
Task Scheduler
```

Permite ejecutar automáticamente:

- scripts;
- copias de seguridad;
- limpieza de archivos;
- tareas de mantenimiento.

La automatización reduce errores y ahorra tiempo.

---

# Buenas prácticas

✔ Mantener Windows actualizado.

✔ Realizar copias de seguridad periódicas.

✔ Supervisar los registros del sistema.

✔ Aplicar el principio de mínimo privilegio.

✔ Documentar los cambios realizados.

✔ Automatizar tareas repetitivas cuando sea posible.

✔ Comprobar periódicamente el espacio disponible.

# 🐧 Linux vs 🪟 Windows

| GNU/Linux | Windows Server |
|------------|----------------|
| SSH | RDP / PowerShell Remoting |
| journalctl | Event Viewer |
| cron | Task Scheduler |
| rsync / tar | Windows Server Backup |
| apt upgrade | Windows Update |
| top / htop | Task Manager |

🎯 La filosofía es idéntica: mantener el servidor estable, seguro y disponible.

- Event Viewer
- Windows Update

# 🖥️ Laboratorio recomendado

En una máquina virtual:

1. Abrir el Visor de eventos.
2. Localizar los registros del sistema.
3. Comprobar Windows Update.
4. Abrir el Programador de tareas.
5. Identificar una tarea programada.
6. Ejecutar:

```powershell
Get-Service
```

7. Comprobar el estado de los servicios principales.

---

# ⚠️ Errores frecuentes

❌ Reiniciar el servidor sin investigar el problema.

✔ Analizar primero los registros y el estado del sistema.

---

❌ No verificar las copias de seguridad.

✔ Una copia debe probarse periódicamente.

---

❌ Instalar actualizaciones en producción sin planificación.

✔ Programar ventanas de mantenimiento.

---

❌ Trabajar siempre mediante RDP.

✔ PowerShell resulta mucho más eficiente para tareas repetitivas.
