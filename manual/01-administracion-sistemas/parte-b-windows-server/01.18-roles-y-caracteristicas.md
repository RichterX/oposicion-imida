# 🧠 ¿Por qué existen los roles?

No todos los servidores realizan las mismas funciones.

Un servidor puede actuar como:

- servidor DNS;
- servidor DHCP;
- servidor web;
- servidor de archivos;
- servidor de impresión;
- controlador de dominio.

En lugar de instalar todas estas funciones por defecto, Windows Server permite añadir únicamente aquellas que sean necesarias.

Esto reduce:

- consumo de recursos;
- superficie de ataque;
- complejidad de administración.

---

# ¿Qué es un rol?

Un **rol** es un conjunto de servicios que permite al servidor desempeñar una función concreta dentro de la red.

Cuando instalamos un rol, Windows incorpora todos los componentes necesarios para esa función.

Ejemplos de roles:

- Active Directory Domain Services (AD DS)
- DNS Server
- DHCP Server
- File and Storage Services
- Web Server (IIS)
- Hyper-V
- Print and Document Services

🎯 **Idea clave**

Un rol define **qué hace** el servidor.

---

# ¿Qué es una característica (Feature)?

Una **característica** es un componente adicional que amplía las capacidades del sistema operativo, pero que no convierte al servidor en un servicio específico.

Ejemplos:

- .NET Framework
- Windows PowerShell
- Telnet Client
- Failover Clustering

🎯 **Idea clave**

Una característica añade funcionalidades; un rol proporciona un servicio.

---

# Server Manager

La principal herramienta gráfica para administrar Windows Server es:

```text
Server Manager
```

Desde ella podemos:

- agregar o quitar roles;
- instalar características;
- administrar servidores remotos;
- revisar eventos;
- comprobar el estado del sistema.

Es el centro de administración de Windows Server.

---

# Instalación de roles

La instalación se realiza mediante el asistente:

```text
Manage

↓

Add Roles and Features
```

El asistente guía al administrador en todo el proceso.

Durante la instalación, Windows puede solicitar automáticamente las características necesarias para el rol seleccionado.

---

# Administración mediante PowerShell

PowerShell permite realizar las mismas tareas desde la línea de comandos.

Consultar los roles instalados:

```powershell
Get-WindowsFeature
```

Instalar un rol:

```powershell
Install-WindowsFeature
```

Desinstalar un rol:

```powershell
Uninstall-WindowsFeature
```

PowerShell resulta especialmente útil para automatizar despliegues y administrar múltiples servidores.

---

# Roles más habituales

| Rol | Función principal |
|------|-------------------|
| AD DS | Controlador de dominio |
| DNS | Resolución de nombres |
| DHCP | Asignación automática de IP |
| IIS | Servidor web |
| File Services | Compartición de archivos |
| Hyper-V | Virtualización |
| Print Services | Gestión de impresoras |

Estos roles serán estudiados con mayor profundidad en capítulos posteriores.

---

# Administración del servidor

Una vez instalado un rol será necesario:

- supervisar su funcionamiento;
- revisar eventos;
- aplicar actualizaciones;
- realizar copias de seguridad;
- comprobar el estado de los servicios asociados.

La instalación de un rol es solo el primer paso de su administración.

---

# 🐧 Linux vs 🪟 Windows

| GNU/Linux | Windows Server |
|------------|----------------|
| `apt install apache2` | Instalar rol IIS |
| `apt install bind9` | Instalar rol DNS |
| `apt install isc-dhcp-server` | Instalar rol DHCP |
| Servicios individuales | Roles integrados |
| `systemctl` | Server Manager + Services |

🎯 Ambos sistemas ofrecen los mismos servicios, aunque el mecanismo de instalación es diferente.

# 🖥️ Laboratorio recomendado (máquina virtual)

1. Abrir **Server Manager**.
2. Acceder al asistente **Add Roles and Features**.
3. Identificar los principales roles disponibles.
4. Ejecutar en PowerShell:

```powershell
Get-WindowsFeature
```

5. Localizar el rol **DNS Server**.
6. Comprobar si está instalado.

No instales todavía ningún rol si esa máquina virtual va a utilizarse para otras prácticas.

---

# ⚠️ Errores frecuentes

❌ Instalar todos los roles "por si acaso".

✔ Instalar únicamente los necesarios.

---

❌ Confundir un rol con una característica.

✔ Un rol proporciona un servicio; una característica añade funcionalidades.

---

❌ Pensar que instalar un rol es suficiente.

✔ También debe mantenerse, supervisarse y actualizarse.
