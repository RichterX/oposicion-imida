# 🧠 ¿Por qué existe NTFS?

Un servidor almacena información crítica.

No basta con guardar archivos.

También debe:

- protegerlos;
- controlar quién accede;
- registrar cambios;
- soportar discos muy grandes;
- recuperarse de errores.

NTFS fue diseñado precisamente para cubrir estas necesidades.

---

# ¿Qué es NTFS?

NTFS (**New Technology File System**) es el sistema de archivos utilizado por defecto en Windows Server y en las versiones modernas de Windows.

Permite almacenar archivos y carpetas incorporando características avanzadas como:

- permisos;
- cifrado;
- compresión;
- cuotas de disco;
- journaling.

---

# FAT32 vs NTFS

| FAT32 | NTFS |
|---------|------|
| Sin permisos avanzados | Permisos ACL |
| Máximo 4 GB por archivo | Archivos muy grandes |
| Sin journaling | Journaling |
| Menor seguridad | Mayor seguridad |
| Uso doméstico | Uso profesional |

🎯 En un servidor Windows prácticamente siempre se utiliza NTFS.

---

# Características principales

NTFS incorpora:

- permisos de seguridad (ACL);
- cifrado mediante EFS;
- compresión;
- cuotas por usuario;
- recuperación ante fallos (journaling);
- soporte para discos y archivos de gran tamaño.

---

# Permisos NTFS

Los permisos determinan qué acciones puede realizar un usuario sobre un archivo o carpeta.

Los permisos básicos más habituales son:

| Permiso | Función |
|----------|---------|
| Control total | Todas las acciones |
| Modificar | Leer, escribir y eliminar |
| Leer y ejecutar | Leer y ejecutar programas |
| Mostrar contenido | Ver carpetas |
| Lectura | Solo lectura |
| Escritura | Crear y modificar |

---

# Herencia de permisos

Por defecto, una carpeta transmite sus permisos a los elementos que contiene.

```text
D:\Investigacion

↓

ProyectoA

↓

datos.xlsx
```

Si no se modifica la configuración, todos heredarán los mismos permisos.

La herencia simplifica enormemente la administración.

---

# Permisos efectivos

Un usuario puede pertenecer a varios grupos.

Windows calcula automáticamente los permisos finales aplicables.

A este resultado se le denomina:

> **Permisos efectivos**

Siempre conviene comprobarlos antes de asumir que un usuario puede acceder a un recurso.

---

# Permisos NTFS y permisos compartidos

Cuando una carpeta se comparte por red intervienen dos niveles de permisos:

```text
Permisos compartidos

+

Permisos NTFS
```

🎯 **Regla importante**

Siempre prevalece el permiso más restrictivo.

Ejemplo:

Compartido → Control total

NTFS → Lectura

Resultado final:

👉 El usuario solo podrá leer.

# ⚠️ Errores frecuentes

❌ Pensar que FAT32 es adecuado para servidores.

✔ NTFS es el sistema recomendado.

---

❌ Confundir permisos compartidos con permisos NTFS.

✔ Ambos se combinan.

---

❌ Desactivar la herencia sin conocer sus consecuencias.

✔ Puede complicar enormemente la administración.
