# 🧠 ¿Por qué existen los usuarios?

Al igual que en GNU/Linux, Windows necesita identificar quién utiliza el sistema.

Cada usuario posee:

- una identidad;
- unas credenciales;
- un perfil;
- unos permisos;
- una pertenencia a grupos.

Gracias a ello es posible controlar quién puede acceder a cada recurso.

---

# Usuarios locales

Un usuario local existe únicamente dentro del propio equipo.

Ejemplos:

- Administrator
- Invitado (Guest)
- ana
- carlos

Estos usuarios solo pueden autenticarse en ese servidor.

Si el equipo no pertenece a un dominio, todas las cuentas serán locales.

---

# Administrator

Windows incorpora una cuenta especial denominada:

```text
Administrator
```

Equivale al usuario **root** en Linux.

Características:

- control total del sistema;
- administración completa;
- instalación de software;
- creación de usuarios;
- modificación de permisos.

⚠️ Como norma general, no debe utilizarse para el trabajo diario.

---

# Perfiles de usuario

Cada usuario dispone de un perfil donde Windows almacena:

- Escritorio;
- Documentos;
- Descargas;
- Configuración personal;
- Preferencias de aplicaciones.

Normalmente se encuentra en:

```text
C:\Users\<usuario>
```

Ejemplo:

```text
C:\Users\ana
```

---

# Grupos

Los grupos permiten administrar permisos de forma conjunta.

En lugar de conceder permisos usuario por usuario, se asignan al grupo.

Después se añaden los usuarios correspondientes.

Ejemplo:

Grupo:

```text
Investigacion
```

Usuarios:

- Ana
- Carlos
- María

Todos heredarán los permisos asignados al grupo.

---

# Tipos de grupos

En un equipo independiente trabajaremos principalmente con:

- Administradores
- Usuarios
- Usuarios avanzados
- Operadores de copia de seguridad
- Escritorio remoto

Cuando estudiemos Active Directory aparecerán nuevos tipos de grupos.

---

# Administración de usuarios

Windows ofrece varias formas de administrar usuarios.

## Usuarios y grupos locales

Ejecutar:

```text
lusrmgr.msc
```

Desde esta consola podemos:

- crear usuarios;
- eliminar usuarios;
- modificar contraseñas;
- administrar grupos.

Es la herramienta más utilizada en servidores independientes.

---

## Administración de equipos

También es posible acceder mediante:

```text
compmgmt.msc
```

↓

Usuarios y grupos locales

---

## PowerShell

Windows permite administrar usuarios mediante PowerShell.

Ejemplo:

Crear un usuario local:

```powershell
New-LocalUser
```

Consultar usuarios:

```powershell
Get-LocalUser
```

Consultar grupos:

```powershell
Get-LocalGroup
```

Añadir un usuario a un grupo:

```powershell
Add-LocalGroupMember
```

No es necesario memorizar todos los comandos, pero conviene conocer su existencia.

---

# Buenas prácticas

✔ Utilizar grupos para asignar permisos.

✔ Evitar utilizar la cuenta Administrator para el trabajo diario.

✔ Eliminar cuentas que ya no se utilicen.

✔ Utilizar contraseñas robustas.

✔ Aplicar el principio de mínimo privilegio.

# 🐧 Linux vs 🪟 Windows

| GNU/Linux | Windows Server |
|------------|----------------|
| root | Administrator |
| /home | C:\Users |
| useradd | New-LocalUser |
| groupadd | New-LocalGroup |
| groups | Get-LocalGroup |
| id | Get-LocalUser |

🎯 Aunque cambian las herramientas, la filosofía es prácticamente la misma.

# ⚠️ Errores frecuentes

❌ Asignar permisos directamente a usuarios.

✔ Es preferible asignarlos a grupos.

---

❌ Trabajar siempre con Administrator.

✔ Utilizar cuentas administrativas únicamente cuando sea necesario.

---

❌ Eliminar un usuario pensando que desaparecerán automáticamente todos sus permisos.

✔ Revisar previamente la pertenencia a grupos y los recursos asociados.

# 📝 Resumen
