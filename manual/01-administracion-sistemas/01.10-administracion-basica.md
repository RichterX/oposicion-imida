# Primer paso: ¿Quién soy y dónde estoy?

Lo primero que hace un administrador nunca es modificar nada.

Primero observa.

```bash
whoami
```

Muestra el usuario actual.

---

```bash
hostname
```

Nombre del equipo.

---

```bash
pwd
```

Directorio actual.

---

```bash
uname -a
```

Información del Kernel.

Ejemplo:

```text
Linux servidor 6.8.0 x86_64 GNU/Linux
```

---

```bash
uptime
```

Tiempo que lleva encendido el servidor.

Ejemplo.

```text
up 27 days
```

🎯 Muy útil para comprobar reinicios inesperados.

# Conexión remota mediante SSH

En un entorno profesional rara vez administraremos el servidor físicamente.

Lo habitual es conectarse por red.

```bash
ssh ana@192.168.1.20
```

Significa:

- Usuario → ana
- Servidor → 192.168.1.20

Una vez autenticados...

Trabajamos como si estuviéramos delante del servidor.

---

# Copiar archivos

## SCP

Copiar un archivo al servidor.

```bash
scp informe.pdf ana@192.168.1.20:/home/ana
```

Copiar desde el servidor.

```bash
scp ana@192.168.1.20:/home/ana/informe.pdf .
```

---

## SFTP

También podemos abrir una sesión interactiva.

```bash
sftp ana@192.168.1.20
```

Muy parecido a un cliente FTP.

---

# Espacio en disco

Uno de los primeros problemas de un servidor suele ser quedarse sin espacio.

Consultar discos.

```bash
df -h
```

Salida típica.

```text
Filesystem      Size Used Avail Use%

/dev/sda1        50G  21G   27G  44%
```

La opción:

```text
-h
```

significa:

> Human readable.

---

# Tamaño de directorios

Queremos saber qué ocupa más espacio.

```bash
du -sh /var/log
```

Ejemplo.

```text
1.2G
```

---

# Memoria RAM

```bash
free -h
```

Ejemplo.

```text
Mem:

8 Gi

2 Gi usados

5 Gi libres
```

---

# Información del sistema

Consultar dispositivos de bloques.

```bash
lsblk
```

Consultar sistemas de archivos montados.

```bash
mount
```

---

# Copias de seguridad

Linux incorpora una herramienta clásica.

```bash
tar
```

Crear una copia.

```bash
tar -czf backup.tar.gz /home/ana
```

Opciones.

```text
c → crear

z → comprimir

f → fichero
```

Restaurar.

```bash
tar -xzf backup.tar.gz
```

---

# Programación de tareas

Muchas tareas deben ejecutarse automáticamente.

Para ello utilizamos:

```bash
cron
```

Editar tareas.

```bash
crontab -e
```

Ver tareas.

```bash
crontab -l
```

Ejemplo.

```text
0 2 * * * backup.sh
```

Significa.

Todos los días.

A las 02:00.

---

# Logs

Cuando ocurre un problema...

Lo primero que consultamos son los registros.

Linux almacena la mayoría en:

```text
/var/log
```

Ejemplos.

```text
syslog

auth.log

kern.log
```

En sistemas modernos también utilizamos.

```bash
journalctl
```

Consultar los últimos mensajes.

```bash
journalctl -n 30
```

---

# Buscar información

Una herramienta imprescindible.

```bash
grep
```

Ejemplo.

Buscar errores.

```bash
grep ERROR archivo.log
```

Buscar PostgreSQL.

```bash
ps aux | grep postgres
```

# Linux en acción

Acabas de recibir un servidor.

Tu checklist sería.

```bash
whoami

hostname

uptime

df -h

free -h

systemctl status postgresql

journalctl -u postgresql

crontab -l
```

En menos de cinco minutos tendrás una idea bastante precisa del estado del sistema.

# Errores frecuentes

❌ No comprobar el espacio en disco.

---

❌ No revisar los logs antes de reiniciar un servicio.

---

❌ No verificar las copias de seguridad.

Una copia que nunca se prueba puede no servir cuando realmente haga falta.

---

❌ Editar `crontab` manualmente.

Siempre utilizar:

```bash
crontab -e
```
