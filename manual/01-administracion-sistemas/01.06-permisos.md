# ¿Por qué existen los permisos?

Imagina un servidor con cien usuarios.

Sin permisos...

- cualquiera podría borrar los archivos de cualquiera;
- cualquiera podría modificar la base de datos;
- cualquiera podría apagar servicios.

Sería un desastre.

Linux protege todos los recursos mediante permisos.

🎯 **Idea clave**

> Linux decide qué puede hacer un usuario con cada archivo o directorio.

---

# Los tres niveles de permisos

Cada archivo tiene permisos para tres tipos de usuarios:

```text
Usuario (Owner)

↓

Grupo (Group)

↓

Otros (Others)
```

No existen más niveles.

Todo Linux gira alrededor de estos tres.

# Linux en acción

Ejecutamos:

```bash
ls -l
```

Obtenemos:

```text
-rwxr-xr--
```

Vamos a dividirlo.

```text
-rwx r-x r--
```

Y ahora.

```text
- | rwx | r-x | r--

    │      │      │

 Usuario Grupo Otros
```

---

# ¿Qué significa la primera letra?

```text
-
```

Indica el tipo de fichero.

| Símbolo | Significado |
|----------|-------------|
| - | Archivo |
| d | Directorio |
| l | Enlace simbólico |

Por ahora nos centraremos en:

- Archivo (`-`)
- Directorio (`d`)

---

# Los permisos

Cada bloque contiene tres letras.

```text
rwx
```

Significan:

| Letra | Significado |
|--------|-------------|
| r | Read (leer) |
| w | Write (escribir) |
| x | Execute (ejecutar) |

---

# Ejemplo completo

```text
-rwxr-xr--
```

Se interpreta así.

## Propietario

```text
rwx
```

Puede:

✅ Leer

✅ Escribir

✅ Ejecutar

---

## Grupo

```text
r-x
```

Puede:

✅ Leer

❌ Escribir

✅ Ejecutar

---

## Otros

```text
r--
```

Puede:

✅ Leer

❌ Escribir

❌ Ejecutar

# chmod

El comando más utilizado para modificar permisos.

Ejemplo.

```bash
chmod u+x script.sh
```

¿Qué significa?

- u → usuario
- + → añadir
- x → ejecución

Resultado.

El propietario podrá ejecutar el archivo.

---

Otro ejemplo.

```bash
chmod g-w proyecto.csv
```

El grupo pierde permiso de escritura.

---

Otro.

```bash
chmod o+r informe.txt
```

Los demás usuarios podrán leer el archivo.

---

# Notación numérica

Linux también utiliza números.

| Valor | Permiso |
|--------|----------|
| 4 | r |
| 2 | w |
| 1 | x |

Se suman.

| Valor | Permiso |
|--------|----------|
| 7 | rwx |
| 6 | rw- |
| 5 | r-x |
| 4 | r-- |

Ejemplo.

```bash
chmod 755 script.sh
```

Equivale a:

```text
rwx r-x r-x
```

Otro.

```bash
chmod 644 informe.txt
```

Equivale a:

```text
rw- r-- r--
```

🎯 **Muy preguntable.**

---

# chown

Permite cambiar el propietario.

```bash
sudo chown ana informe.txt
```

Ahora Ana será la propietaria.

También puede cambiar propietario y grupo.

```bash
sudo chown ana:investigacion informe.txt
```

---

# chgrp

Permite cambiar únicamente el grupo.

```bash
sudo chgrp investigacion informe.txt
```

---

# umask

Cuando se crea un archivo...

Linux no empieza desde cero.

Aplica una máscara llamada:

```text
umask
```

Ejemplo habitual.

```bash
umask
```

Salida.

```text
0022
```

Esto provoca normalmente:

Archivos nuevos.

```text
644
```

Directorios nuevos.

```text
755
```

No necesitas calcular máscaras complicadas.

Solo recordar este comportamiento.

---

# Linux en acción

Creamos un archivo.

```bash
touch informe.txt
```

Consultamos.

```bash
ls -l
```

Salida.

```text
-rw-r--r--
```

Pregunta.

¿Quién puede modificarlo?

Respuesta.

Solo el propietario.

# Errores frecuentes

❌ Confundir propietario con grupo.

---

❌ Pensar que 777 es una buena práctica.

No lo es.

Otorga permisos completos a todos.

---

❌ Utilizar chmod sin comprobar los permisos actuales.

Siempre conviene ejecutar antes:

```bash
ls -l
```
