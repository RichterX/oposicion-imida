# ¿Qué es Bash?

Bash significa:

**Bourne Again SHell**

Es el intérprete de comandos más utilizado en sistemas GNU/Linux.

La Shell actúa como intermediaria entre el usuario y el Kernel.

```text
Usuario

↓

Bash

↓

Kernel

↓

Hardware
```

🎯 **Idea clave**

> El Kernel no habla directamente con el usuario.

> Necesita una Shell.

---

# Linux en acción

Escribimos:

```bash
pwd
```

¿Qué ocurre?

1. Bash recibe el comando.
2. Busca el ejecutable correspondiente.
3. Solicita al Kernel que lo ejecute.
4. El Kernel realiza la operación.
5. Bash muestra el resultado.

---

# Variables de entorno

Bash almacena información en variables.

Podemos consultar una variable con:

```bash
echo $HOME
```

Salida:

```text
/home/admin
```

Otras variables importantes:

```bash
echo $USER
```

```bash
echo $PATH
```

```bash
echo $PWD
```

| Variable | Significado |
|-----------|-------------|
| HOME | Directorio personal |
| USER | Usuario actual |
| PATH | Rutas de búsqueda de comandos |
| PWD | Directorio actual |

---

# PATH

Cuando escribimos:

```bash
ls
```

¿Por qué Bash encuentra ese programa?

Porque busca en las rutas almacenadas en:

```bash
echo $PATH
```

Ejemplo:

```text
/usr/local/bin
/usr/bin
/bin
```

---

# Redirecciones

Podemos guardar la salida de un comando.

```bash
ls > archivos.txt
```

Ahora la salida ya no aparece por pantalla.

Se guarda en el fichero.

---

Añadir sin sobrescribir.

```bash
ls >> archivos.txt
```

---

Entrada desde un archivo.

```bash
comando < fichero.txt
```

---

# Pipes

Uno de los conceptos más potentes de Linux.

Permiten conectar comandos.

Ejemplo.

```bash
ps aux | grep postgres
```

¿Qué ocurre?

```text
ps aux

↓

lista procesos

↓

grep postgres

↓

solo muestra PostgreSQL
```

🎯 Es uno de los comandos más utilizados por administradores.

---

Otro ejemplo.

```bash
ls -l | less
```

---

# Alias

Podemos crear atajos.

```bash
alias ll="ls -la"
```

Ahora:

```bash
ll
```

equivale a:

```bash
ls -la
```

---

# Scripts

Un script Bash es simplemente un fichero con comandos.

Ejemplo.

```bash
#!/bin/bash

echo "Buenos días"

date

whoami
```

Guardamos:

```text
saludo.sh
```

Damos permisos.

```bash
chmod +x saludo.sh
```

Ejecutamos.

```bash
./saludo.sh
```

---

# Linux en acción

Queremos saber si PostgreSQL está funcionando.

```bash
ps aux | grep postgres
```

Después conocer el usuario.

```bash
whoami
```

Y guardar la fecha.

```bash
date >> registro.log
```

Todo puede automatizarse en un script.

❌ Confundir Shell con Kernel.

---

❌ Olvidar el símbolo `$` al consultar variables.

---

❌ Pensar que un Pipe guarda información.

No la guarda.

Simplemente conecta la salida de un comando con la entrada del siguiente.

---

❌ Ejecutar un script sin permiso de ejecución.
