# ¿Por dónde empezar?

Antes de crear un usuario...

Primero debemos comprobar si ya existe.

## Comando

```bash
id ana
```

Si el usuario existe obtendremos algo parecido a:

```text
uid=1001(ana) gid=1001(ana) groups=1001(ana)
```

Si no existe:

```text
id: 'ana': no existe el usuario
```

🎯 **Idea importante**

No intentes crear un usuario sin comprobar antes si ya existe.

---

# Crear un usuario

El comando básico es:

```bash
sudo useradd ana
```

¿Qué hace?

- Crea el usuario.
- Asigna un UID.
- Crea un grupo principal (según la configuración del sistema).
- Registra la información en los ficheros correspondientes.

⚠️ **Importante**

En muchas distribuciones **NO crea automáticamente el directorio personal**.

Para hacerlo:

```bash
sudo useradd -m ana
```

La opción `-m` crea:

```text
/home/ana
```

Esta diferencia aparece con frecuencia en entrevistas técnicas y conviene recordarla.

---

# Asignar una contraseña

Después de crear un usuario normalmente hay que asignarle una contraseña.

```bash
sudo passwd ana
```

El sistema solicitará:

```text
Nueva contraseña:
Repetir contraseña:
```

Hasta que no tenga contraseña, el usuario normalmente no podrá autenticarse mediante contraseña.

---

# Consultar información de un usuario

Podemos utilizar:

```bash
id ana
```

o

```bash
groups ana
```

Ejemplo:

```text
ana : ana investigacion
```

Esto significa que Ana pertenece a:

- Grupo principal → ana
- Grupo secundario → investigacion

---

# Cambiar de usuario

En ocasiones un administrador necesita comprobar qué puede hacer otro usuario.

Para iniciar una sesión como otro usuario utilizamos:

```bash
su - ana
```

La opción `-` inicia una sesión completa cargando el entorno del usuario (HOME, PATH y variables de entorno).

Si queremos volver al usuario anterior:

```bash
exit
```

También es posible ejecutar un único comando como otro usuario:

```bash
sudo -u ana whoami
```

Salida:

```text
ana
```

💡 **Consejo**

Cuando estés administrando un servidor, es habitual utilizar `su - usuario` para comprobar si un problema se reproduce realmente con ese usuario.

# 💡 Curiosidad

En distribuciones como Debian y Ubuntu también existe el comando:

```bash
adduser
```

Se trata de una herramienta más amigable que guía paso a paso la creación del usuario.

Internamente utiliza `useradd`.

Para esta oposición nos centraremos en **useradd**, ya que es el comando estándar presente en prácticamente todas las distribuciones Linux.

---

# Crear un grupo

```bash
sudo groupadd investigacion
```

Ahora existe un grupo llamado:

```text
investigacion
```

---

# Añadir un usuario a un grupo

```bash
sudo usermod -aG investigacion ana
```

Esta es probablemente la opción más preguntable del capítulo.

Veámosla despacio.

`-G`

Lista de grupos suplementarios.

`-a`

Append.

Añade el grupo sin eliminar los existentes.

⚠️ Si olvidamos `-a` podremos eliminar accidentalmente la pertenencia a otros grupos.

Esta es una trampa clásica.

---

# Modificar un usuario

El comando general es:

```bash
sudo usermod
```

Ejemplos.

Cambiar el directorio personal:

```bash
sudo usermod -d /datos/ana ana
```

Cambiar el nombre del usuario:

```bash
sudo usermod -l investigadora ana
```

---

# Eliminar un usuario

```bash
sudo userdel ana
```

El usuario desaparece.

Pero...

Su directorio personal puede permanecer.

Para eliminar también el directorio:

```bash
sudo userdel -r ana
```

La opción `-r` elimina:

- usuario
- directorio personal
- correo local

---

# ¿Dónde guarda Linux esta información?

Cuando ejecutas:

```bash
useradd
```

Linux modifica principalmente:

```text
/etc/passwd
```

Y también:

```text
/etc/shadow
```

y

```text
/etc/group
```

No necesitas memorizar todavía el formato de estos archivos.

Solo debes recordar que existen.

Los estudiaremos más adelante.

# Linux en acción

Imagina que debes preparar el servidor para tres investigadores.

```bash
sudo groupadd investigacion

sudo useradd -m ana
sudo passwd ana
sudo usermod -aG investigacion ana

sudo useradd -m carlos
sudo passwd carlos
sudo usermod -aG investigacion carlos

sudo useradd -m maria
sudo passwd maria
sudo usermod -aG investigacion maria
```

Al finalizar...

Todos pertenecen al grupo:

```text
investigacion
```

Y podrán compartir recursos.

Si eres capaz de completar este laboratorio sin consultar el manual, puedes considerar dominado este capítulo.

# ❌ Errores frecuentes

❌ Olvidar `-m`.

Resultado:

El usuario no tiene directorio personal.

---

❌ Utilizar `usermod -G` sin `-a`.

Resultado:

Se sustituyen los grupos existentes.

---

❌ Eliminar un usuario sin `-r`.

Resultado:

Su directorio personal permanece en el sistema.
