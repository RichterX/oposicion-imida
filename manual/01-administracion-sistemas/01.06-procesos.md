# ¿Qué es un proceso?

Un **programa** es un conjunto de instrucciones almacenadas en disco.

Por ejemplo:

```text
Firefox
PostgreSQL
Apache
Python
```

Cuando uno de esos programas comienza a ejecutarse...

Linux crea un **proceso**.

🎯 **Idea clave**

> Programa = archivo.

> Proceso = programa en ejecución.

---

# ¿Qué información tiene un proceso?

Cada proceso posee, entre otros datos:

- PID
- Usuario propietario
- Memoria utilizada
- Tiempo de CPU
- Prioridad
- Estado

---

# PID

Cada proceso tiene un identificador único.

Se llama:

```text
PID
```

(Process Identifier)

Ejemplo:

```text
PID 1204

apache2
```

Nunca existen dos procesos con el mismo PID al mismo tiempo.

---

# Linux en acción

Consultamos los procesos.

```bash
ps
```

Salida:

```text
 PID TTY          TIME CMD

1234 pts/0    00:00:00 bash

1258 pts/0    00:00:00 ps
```

Solo vemos los procesos asociados a nuestra terminal.

---

# Ver todos los procesos

Lo habitual es utilizar:

```bash
ps aux
```

Ejemplo:

```text
USER       PID %CPU %MEM COMMAND

root         1  0.0  0.1 systemd

postgres   824  0.2  0.4 postgres

www-data   910  0.0  0.3 apache2
```

🎯 Es uno de los comandos más utilizados por un administrador.

---

# top

Para observar el sistema en tiempo real.

```bash
top
```

Muestra continuamente:

- CPU
- Memoria
- Procesos
- Carga del sistema

Es como un monitor en directo.

---

# htop

Versión mejorada de `top`.

```bash
htop
```

Más visual.

Permite navegar con el teclado.

No siempre viene instalado.

---

# Finalizar procesos

A veces un programa deja de responder.

Podemos finalizarlo.

```bash
kill 1258
```

1258 es el PID.

Linux enviará una señal al proceso.

---

# Señales

La más habitual:

```bash
kill -15 1258
```

Solicita al proceso que termine correctamente.

Si no responde:

```bash
kill -9 1258
```

Fuerza la finalización inmediata.

⚠️ **No abuses de `kill -9`.**

Siempre que sea posible utiliza primero `kill` o `kill -15`.

---

# Prioridad de ejecución

Linux permite indicar qué procesos son más importantes.

Aquí aparecen dos conceptos.

## nice

Inicia un proceso con una prioridad determinada.

```bash
nice -n 10 programa
```

---

## renice

Modifica la prioridad de un proceso existente.

```bash
renice 5 1258
```

---

# Procesos en segundo plano

Podemos ejecutar:

```bash
sleep 100 &
```

El símbolo:

```text
&
```

envía el proceso al segundo plano.

---

# jobs

Permite ver los procesos iniciados desde la Shell actual.

```bash
jobs
```

Ejemplo.

```text
[1]+ Running sleep 100 &
```

---

# fg

Trae un proceso al primer plano.

```bash
fg
```

---

# bg

Envía un proceso suspendido al segundo plano.

```bash
bg
```

# Linux en acción

Supongamos que PostgreSQL deja de responder.

Primero comprobamos:

```bash
ps aux | grep postgres
```

Encontramos:

```text
824 postgres
```

Intentamos cerrar correctamente.

```bash
kill 824
```

Si continúa bloqueado.

```bash
kill -9 824
```

---

# Errores frecuentes

❌ Confundir programa con proceso.

---

❌ Utilizar siempre `kill -9`.

---

❌ Pensar que `ps` muestra todos los procesos.

Solo muestra los asociados a la terminal actual.
