# 🧪 Misiones

> "La teoría te dice qué hace un comando. Las misiones te enseñan cuándo utilizarlo."

---

# ¿Qué son las misiones?

Las misiones son escenarios prácticos diseñados para aplicar los conocimientos adquiridos en el manual.

No forman parte del temario oficial.

Su objetivo es desarrollar la capacidad de razonamiento y la toma de decisiones ante problemas reales de administración de sistemas.

Cada misión integra conceptos de varios capítulos y simula situaciones similares a las que puede encontrar un administrador de sistemas en su trabajo diario.

---

# Objetivos

Las misiones persiguen varios objetivos:

- Aplicar los conocimientos adquiridos.
- Relacionar conceptos entre diferentes capítulos.
- Aprender a diagnosticar problemas.
- Elegir la herramienta adecuada en cada situación.
- Detectar puntos débiles antes de realizar un simulacro.

---

# Metodología

Cada misión presenta un escenario realista.

A lo largo de la misión aparecerán diferentes incidencias que deberán resolverse utilizando los conocimientos adquiridos durante el estudio del bloque correspondiente.

Las respuestas no consisten únicamente en recordar comandos.

También es importante justificar las decisiones tomadas.

---

# Niveles de dificultad

Las misiones están divididas en varios niveles.

| Nivel | Objetivo |
|--------|----------|
| 🟢 Básico | Comprobar conocimientos fundamentales. |
| 🟡 Intermedio | Resolver incidencias sencillas. |
| 🟠 Avanzado | Relacionar varios conceptos del bloque. |
| 🔴 Experto | Resolver incidencias complejas similares a un entorno real. |

Cada nivel aumenta progresivamente la dificultad.

---

# Corrección

Cada misión dispone de dos documentos:

```text
mision-XX.md
```

Contiene únicamente el enunciado.

Y

```text
mision-XX-soluciones.md
```

Incluye:

- solución recomendada;
- alternativas válidas;
- explicación técnica;
- errores habituales;
- buenas prácticas de administración.

Las soluciones se construirán de forma iterativa durante el desarrollo del proyecto.

El orden recomendado de estudio es el siguiente:

```text

↓

Tarjetas

↓
```

---

# Filosofía

Estas misiones no pretenden enseñar a memorizar comandos.

Pretenden enseñar a pensar como un administrador de sistemas.

Ante una incidencia, el objetivo no es recordar automáticamente un comando concreto, sino responder a preguntas como:

- ¿Qué está ocurriendo?
- ¿Qué información necesito?
- ¿Qué herramienta me permite obtener esa información?
- ¿Cuál es la solución más adecuada?

El razonamiento tendrá siempre prioridad sobre la memorización.

# Nota del preparador

Una misión no se considera completada cuando todas las respuestas son correctas.

Se considera completada cuando el alumno comprende por qué cada decisión es la más adecuada.

Ese es el objetivo principal de este proyecto.
